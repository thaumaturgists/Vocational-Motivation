3.1 Modify HTML with JavaScript

Summary & Quick Review

3.1.1 Steps and Methods for Changing HTML “On The Fly”

JavaScript can dynamically alter your webpage’s structure or content by manipulating the DOM once the HTML is loaded.

  • document.getElementById("id"): Selects a single element by its unique id.
  • document.getElementsByName("name"): Selects elements by their name attribute (often used for forms).
  • document.getElementsByTagName("tag"): Returns a live collection of elements that match a specified tag (e.g., div, p).
  • document.getElementsByClassName("class"): Returns a live collection of elements that match a specific CSS class.
  • Modern Approach:
    • document.querySelector("cssSelector"): Returns the first match.
    • document.querySelectorAll("cssSelector"): Returns all matches as a static NodeList.

Once selected, you can modify properties like innerHTML, textContent, and style.

Example:
<div class="message">Hello</div>
            <script>
            let msgDiv = document.querySelector(".message");
            msgDiv.textContent = "Hello, new content!";
            msgDiv.style.color = "blue";
            </script>

Why It Matters: Real-time updates to text, styles, or structure allow for dynamic and interactive user experiences.

3.1.2 Modify Attributes in HTML Using DOM Elements

  • element.setAttribute("attributeName", "value"): Dynamically change or add attributes.
  • element.removeAttribute("attributeName"): Remove an existing attribute.
  • Direct Property Access: For some attributes, you can directly manipulate them (e.g., img.src = "newImage.png";).
Example:
<img id="myImg" src="oldImage.png" alt="Old Image">
            <script>
            let myImg = document.getElementById("myImg");
            myImg.setAttribute("src", "newImage.png");
            myImg.alt = "New Image"; // direct property
            </script>

Pitfalls: Confusing the difference between element.attribute (property access) and element.setAttribute() (DOM attribute-level manipulation). In many cases, they behave the same, but with certain properties (like value vs. defaultValue on form inputs), behavior can differ.

3.1.3 Modify Form Object Values

You can grab input fields (text fields, checkboxes, radio buttons, etc.) and change or read their values in real time.

Example:
<input type="text" id="username" value="Alice">
                <button onclick="changeUser()">Change Username</button>
            <script>
            function changeUser() {
                let userInput = document.getElementById("username");
                userInput.value = "Bob"; // change the text field to "Bob"
            }
            </script>

Why It Matters: Real-time updates to text, styles, or structure allow for dynamic and interactive user experiences.

3.2 Use JavaScript to Develop Interactive Forms

Summary & Quick Review

3.2.1 Identify and Use Form Controls

  • Common form elements:
    • Text Fields: <input type="text">
    • Radio Buttons: <input type="radio">
    • Checkboxes: <input type="checkbox">
    • Select Dropdowns: <select><option>...</option></select>
    • HTML5 additions: <input type="email">, <input type="date">, <input type="range">
  • Why It Matters: HTML5 provides built-in validation (e.g., type="email" ensures valid email format).

3.2.2 Define the Form Object

  • Access forms via document.forms or document.getElementById("formId").

3.2.3 Refer to Form Objects

  • Access form controls by their name attribute:
    • <form id="myForm">
            <input type="text" name="username">
        </form>
        
        let theForm = document.getElementById("myForm");
        console.log(theForm.username.value); // Access input via name
  • Avoid relying on form.elements[0] for robustness.

3.2.4 Use Form Objects

  • Read values: myForm.username.value
  • Set values: myForm.username.value = "Alice"
  • Check status: myForm.termsCheckbox.checked

3.2.5 Conduct Form Validation

  • Client-Side Validation with JavaScript:
    • Check required fields, email format, password complexity, etc.
  • HTML5 Validation attributes: required, pattern, minlength, maxlength
  • Custom Validation Example: 
    function validateForm() {
            let emailField = document.getElementById("email");
            if (!emailField.value.includes("@")) {
                alert("Please enter a valid email.");
                return false; // stops form submission
            }
            return true;
        }
  • Why Validate on Client-Side?
    • Quick feedback for users, minimal server load.
    • Always combine with server-side validation for security.

3.2.6 Identify Common Form Security Issues

  • Input Injection (XSS, SQL injection if not handled server-side).
  • CSRF (Cross-Site Request Forgery) risks.
  • Over-reliance on Client-Side Validation: Malicious users can bypass it.

3.3 Address JavaScript Security Issues Involving Browsers and Cookies

Summary & Quick Review

3.3.1 Distinguish Between Browser and OS Security

  • Browser Security: Sandboxes JavaScript from accessing OS resources directly.
  • Operating System Security: Manages deeper system-level operations, but JavaScript has no direct access in web scenarios.

3.3.2 Browser Security Issues Relevant to JavaScript

  • Script Blocking: Users or extensions can block scripts.
  • Frame-to-Frame URL Changing: Malicious behavior if scripts manipulate iframes outside their domain.
  • document.write(): Can overwrite the entire page if used after the initial load; rarely recommended.

Pitfall: Overusing document.write() can lead to unexpected results.

3.3.3 Signed Scripts

JavaScript can be digitally signed to indicate authenticity, relevant in secure environments.

3.3.4 Client-Side Browser Detection & Compatibility

Use feature detection instead of navigator.userAgent due to potential spoofing.

3.3.5 Common Issues for Creating Secure JavaScript Code

  • Never store sensitive information in plain text.
  • Sanitize user input to prevent injection attacks.
  • Use HTTPS to protect data in transit.
  • Be cautious of third-party scripts.

3.3.6 Define Cross-Site Scripting (XSS)

A vulnerability where attackers inject malicious scripts into trusted content, often from unsanitized user input.

3.3.7 Functions and Common Uses of Cookies

  • Cookies store small data pieces (up to ~4KB) in the browser.
  • Uses include session tracking, remembering preferences, and analytics.

3.3.8 Manipulate Cookies Effectively

  • Creating/Updating a Cookie: document.cookie = "username=Alice; expires=Fri, 31 Dec 9999 23:59:59 GMT; path=/";
  • Retrieving Cookies: document.cookie returns all cookies in key=value pairs.
  • Clearing Cookies: Set the cookie’s expires date to a past date.
  • Enable/Disable Cookies: This is a browser-level setting.

3.3.9 Ethics in Collecting, Storing, Using, and Protecting User Data

  • Consent: Inform users about data collection.
  • Minimize Data: Store only what’s necessary.
  • Security: Use encryption and secure transport (HTTPS).
  • Legal Considerations: Comply with GDPR, CCPA, etc.
